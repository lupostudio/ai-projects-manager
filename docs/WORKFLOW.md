# WORKFLOW.md

## Detailed Real-World Usage Workflows

### Workflow 1: Setting Up the Project
1. **Clone the repository**: Use the command `git clone https://github.com/lupostudio/ai-projects-manager.git` to clone the repository to your local machine.
2. **Navigate to the project directory**: Change to the project folder using `cd ai-projects-manager`.
3. **Install dependencies**: Run `npm install` to install the required packages.

### Workflow 2: Adding a New Feature
1. **Create a new branch**: Use `git checkout -b feature/new-feature` to create and switch to a new branch.
2. **Implement the feature**: Make necessary code changes in your preferred IDE.
3. **Commit your changes**: Use `git add .` and then `git commit -m "Add new feature"` to stage and commit your changes.
4. **Push the branch**: Run `git push origin feature/new-feature` to push your branch to GitHub.

### Workflow 3: Opening a Pull Request
1. **Navigate to the repository on GitHub**: Go to the main page of your repository.
2. **Click on the "Compare & pull request"** button next to your recently pushed branch.
3. **Fill out the pull request form**: Include a title and description of your changes.
4. **Submit Pull Request**: Click the button to create the pull request.

### Workflow 4: Merging Changes
1. **Review Pull Request**: Check the changes and comments from reviewers on the pull request page.
2. **Merge the branch**: If approved, use the "Merge pull request" button on GitHub.
3. **Delete the branch**: Optionally, delete the feature branch after merging by clicking on "Delete branch".
